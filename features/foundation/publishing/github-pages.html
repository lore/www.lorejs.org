Publishing to GitHub Pages

Instructions for publishing to [GitHub Pages](https://pages.github.com).

At the moment, GitHub Pages is more complicated to publish to than Surge, so if you're not comfortable with Gulp or changing DNS settings to create a CNAME, it's recommended you don't use this approach until there is a more streamlined solution in place.

Install the Gulp Task

To publish your project to GitHub Pages first install the gulp task to do so:

{% highlight bash %} lore generate github {% endhighlight %}

This task will automatically install any dependencies and add them to the devDependencies in your package.json.

Publish Your Project

Next you'll need to run the gulp task to publish your project:

{% highlight bash %} gulp github {% endhighlight %}

This task will build your project for production, create branch called gh-pages, and push your code into it.

Once this completes your code will be available at http://[username].github.io/[repository], where username is your GitHub username and repository is the name of your repository. For example, if your username was fluffykittens and you were creating an application called furballs, your application would be available at http://fluffykittens.github.io/furballs.

To fix that, you'll also need to change your routes.js file so that the root route is the repository of your application, like this:

With those changes in place, once you redploy, your application should work on GitHub Pages.

Non-Modification Option

If you don't want to modify your files to be served from /furballs/, your other option is to [create a custom domain](https://help.github.com/articles/using-a-custom-domain-with-github-pages/) so that your application can be served from http://www.furballs.com.

If you want to go this approach and buy a custom domain, and don't already have a DNS provider (or want to switch from the one you currently have), I highly recommend [DNSimple](https://dnsimple.com).
