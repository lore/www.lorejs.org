--- layout: quickstart permalink: /quickstart/fetching/step-4/ --- {% capture markdown %} # Step 4: Display Loading Experience In this step we're going to use the `state` property in the data structure in order to display a loading message while the list of tweets are being fetching. ### The Problem While it may happen too quickly to notice, there is a period of time before the list of tweets is displayed when the page simply says "Feed" with a blank experience underneath. This happens because the application doesn't fetch the tweets until the first time the `Feed` component is rendered, and that component can't render data it doesn't have yet. Showing a blank view is a bad user experience, so let's update our `Feed` component to display the text "Loading..." while the tweets are being fetched. ### Add a Loading Experience When Lore interacts with data (such as fetching, creating, updating or deleting it) the action creators built into the framework update the `state` property of the data to reflect the action being performed. The first time our `Feed` component is rendered for example, it requests the data associated with `tweet.find` using the `getState` method of the `connect` decorator. Since this data doesn't exist yet, the framework invokes an action to go fetch it. This action will set the `state` property of the data to `FETCHING` in order to notify you that the data is being fetched. Once the data returns, the action will update the `state` property to have a value of `RESOLVED` to signify that the data has been fetched. If there is an error fetching the data, the state would be updated to `ERROR_FETCHING`. Let's use this behavior to create our loading experience. We're going to start by importing a file called `PayloadStates` that resides in `src/constants`. This file is the set of string constants that the framework applies to data by default.
