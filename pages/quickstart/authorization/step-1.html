--- layout: quickstart permalink: /quickstart/authorization/step-1/ --- {% capture markdown %} # Step 1: Hide Edit Link In this step we're going to wrap our `EditLink` with a decorator that will only display it for the user who created the tweet. ### The Authorization Decorator If you look inside `src/decorators` you'll see one called `UserIsAuthorized` that looks like this: {% tabs tab1={"name": "ES5", "id": 101} | tab2={"name": "ES6", "id": 102} | tab3={"name": "ESNext", "id": 103} %} {% tab id=101, is_active=true %} ```jsx var React = require('react'); var AuthorizationGenerator = require('lore-auth').AuthorizationGenerator; module.exports = AuthorizationGenerator({ wrapperDisplayName: 'UserIsAuthorized', isAuthorized: function(storeState) { return true; } }); ``` {% endtab %} {% tab id=102 %} ```jsx import React from 'react'; import { AuthorizationGenerator } from 'lore-auth'; export default AuthorizationGenerator({ wrapperDisplayName: 'UserIsAuthorized', isAuthorized(storeState) { return true; } }) ``` {% endtab %} {% tab id=103 %} ```jsx import React from 'react'; import { AuthorizationGenerator } from 'lore-auth'; export default AuthorizationGenerator({ wrapperDisplayName: 'UserIsAuthorized', isAuthorized(storeState) { return true; } }) ``` {% endtab %} {% endtabs %} This decorator is designed to wrap a component, and will only render that component if the `isAuthorized` function returns true. The `isAuthorized` function recieves a copy of the current store state as a convenience, in case you need to inspect it for the current user, the user's permissions, or anything else neccesary to determine authorization. We're going to be using this decorator to hide the edit link from any users who were not the author of the tweet. ### Create the UserCanEditTweet Decorator Create a copy of the `UserIsAuthorized` decorator and rename it to `UserCanEditTweet`. Then update the code to look like this: {% tabs tab1={"name": "ES5", "id": 104} | tab2={"name": "ES6", "id": 105} | tab3={"name": "ESNext", "id": 106} %} {% tab id=104, is_active=true %} ```jsx var React = require('react'); var AuthorizationGenerator = require('lore-auth').AuthorizationGenerator; module.exports = AuthorizationGenerator({ wrapperDisplayName: 'UserCanEditTweet', propTypes: { tweet: React.PropTypes.object.isRequired }, contextTypes: { user: React.PropTypes.object.isRequired }, isAuthorized: function (storeState) { var tweet = this.props.tweet; var user = this.context.user; return tweet.data.userId === user.id; } }); ``` {% endtab %} {% tab id=105 %} ```jsx import React, { PropTypes } from 'react'; import { AuthorizationGenerator } from 'lore-auth'; export default AuthorizationGenerator({ wrapperDisplayName: 'UserCanEditTweet', propTypes: { tweet: PropTypes.object.isRequired }, contextTypes: { user: React.PropTypes.object.isRequired }, isAuthorized(storeState) { var tweet = this.props.tweet; var user = this.context.user; return tweet.data.userId === user.id; } }) ``` {% endtab %} {% tab id=106 %} ```jsx import React, { PropTypes } from 'react'; import { AuthorizationGenerator } from 'lore-auth'; export default AuthorizationGenerator({ wrapperDisplayName: 'UserCanEditTweet', propTypes: { tweet: PropTypes.object.isRequired }, contextTypes: { user: React.PropTypes.object.isRequired }, isAuthorized(storeState) { var tweet = this.props.tweet; var user = this.context.user; return tweet.data.userId === user.id; } }) ``` {% endtab %} {% endtabs %} In the code above we're declaring that the decorator expects to receive a tweet. And since we've stored the current user in the context, we're going to retrieve it from there instead of from the store's state directly. In the `isAuthorized` function we are then going to check whether the current user was the author of the tweet. ### Wrap the Edit Link To use this decorator, open the `EditLink` component and decorate the component just like you would when using `lore.connect`. {% tabs tab1={"name": "ES5", "id": 107} | tab2={"name": "ES6", "id": 108} | tab3={"name": "ESNext", "id": 109} %} {% tab id=107, is_active=true %} ```jsx var UserCanEditTweet = require('../decorators/UserCanEditTweet'); ``` {% endtab %} {% tab id=109 %} ```jsx import UserCanEditTweet from '../decorators/UserCanEditTweet'; @UserCanEditTweet class EditLink extends Component { ... } ``` {% endtab %} {% endtabs %} Now refresh the page and the edit links should disappear from any tweets not created by Ayla. ### Visual Check-in If everything went well, your application should now look like this. ## Code Changes Below is a list of files modified during this step. ### src/decorators/UserCanEditTweet.js {% tabs tab1={"name": "ES5", "id": 1} | tab3={"name": "ES6", "id": 2} | tab4={"name": "ESNext", "id": 3} %} {% tab id=1, is_active=true %} ```jsx var React = require('react');
