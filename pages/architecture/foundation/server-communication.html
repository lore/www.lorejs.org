Core challenge is preventing action/reducer explosion and being able to accommodate different API server conventions. ### Patterns That Lore is Built On The video above talks about the interface Lore exposes for API communication, and how through conventions it can eliminate the need to write actions and reducers for many endpoints. What the video doesn't explain is _how_ the framework does this and what those patterns are. A full explanation of this is not yet recorded, though it does exist in presentation form. Once it's recorded this section will contain that explanation.
