By default it's configured to use the `dialog` DOM element in `index.html`, but can be configured to use something else. ```html
``` The helper method is appended to `lore.dialog`. ## Config ```js module.exports = { /** * DOM Element ID that the dialogs should be mounted to. Should be located * outside the DOM element the application is mounted to. */ // domElementId: 'dialog' }; ``` ## Example Usage Given a dialog called `UpdateTweetDialog`, this hook is used like this: ```js var tweet = this.props.tweet; function updateTweet(params) { lore.actions.tweet.update(tweet, params); } lore.dialog.show(function(){ return ( ); }); ``` The `lore.dialog.show` method makes sure the component gets instantiated and mounted to the correct DOM element. It also makes sure any dialog previously mounted to that DOM element is removed beforehand. {% endcapture %} {{ markdown | markdownify }}
