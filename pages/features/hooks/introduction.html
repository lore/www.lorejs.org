--- layout: features permalink: /features/hooks/introduction/ --- {% capture markdown %} # Hooks The vast majority of functionality in Lore is implemented as a series of plugins calls hooks. This is done to make it easier to modify and extend the functionality of the framework. New projects include a set of default hooks, which you can see if you open up the `index.js` file at the root of your project: ```js lore.summon({ hooks: { auth: require('lore-hook-auth'), actions: require('lore-hook-actions'), bindActions: require('lore-hook-bind-actions'), collections: require('lore-hook-collections'), connections: require('lore-hook-connections'), connect: require('lore-hook-connect'), dialog: require('lore-hook-dialog'), dialogs: require('lore-hook-dialogs-bootstrap'), models: require('lore-hook-models'), react: require('lore-hook-react'), reducers: require('lore-hook-reducers'), redux: _.extend(require('lore-hook-redux'), { dependencies: ['reducers', 'auth'] }), router: require('lore-hook-router') } }); ``` ### List of Default Hooks The following is a short description of what each of the default hooks do. * [lore-hook-actions](https://github.com/lore/lore/tree/master/packages/lore-hook-actions): creates a set of actions for each model from blueprints * [lore-hook-auth](https://github.com/lore/lore/tree/master/packages/lore-hook-auth): provides an action and reducer dedicated to fetching the current user * [lore-hook-bind-actions](https://github.com/lore/lore/tree/master/packages/lore-hook-bind-actions): binds all actions to the dispatch method of the Redux store to make them simpler to invoke * [lore-hook-collections](https://github.com/lore/lore/tree/master/packages/lore-hook-collections): creates a Collection for each model in the application, using information in `connections` config * [lore-hook-connect](https://github.com/lore/lore/tree/master/packages/lore-hook-connect): generates the lore.connect decorator to simplify data retrieval * [lore-hook-connections](https://github.com/lore/lore/tree/master/packages/lore-hook-connections): allows you to describe the APIs your application will be consuming * [lore-hook-dialog](https://github.com/lore/lore/tree/master/packages/lore-hook-dialog): provides a utility for mounting dialogs * [lore-hook-models](https://github.com/lore/lore/tree/master/packages/lore-hook-models): creates a Model for each model in the application, using information in `connections` config * [lore-hook-react](https://github.com/lore/lore/tree/master/packages/lore-hook-react): hook responsible for building and mounting the application to the DOM * [lore-hook-reducers](https://github.com/lore/lore/tree/master/packages/lore-hook-reducers): creates a set of reducers for each model from blueprints * [lore-hook-redux](https://github.com/lore/lore/tree/master/packages/lore-hook-redux): generates the Redux Store from all the reducers * [lore-hook-router](https://github.com/lore/lore/tree/master/packages/lore-hook-router): loads the react-router routes and history type ### Additional Hooks In additional to the default actions, there are other actions you can install on an as-needed basis. * [lore-hook-dialogs-bootstrap](https://github.com/lore/lore/tree/master/packages/lore-hook-dialogs-bootstrap): automatically generates [Bootstrap dialogs](http://getbootstrap.com/javascript/#modals) for each model in the application * [lore-hook-dialogs-material-ui](https://github.com/lore/lore/tree/master/packages/lore-hook-dialogs-material-ui): automatically generates [Material UI dialogs](http://www.material-ui.com/#/components/dialog) for each model in the application * [lore-hook-polling](https://github.com/lore/lore/tree/master/packages/lore-hook-polling): provides the ability to poll actions on an interval * [lore-hook-websockets-actioncable](https://github.com/lore/lore/tree/master/packages/lore-hook-websockets-actioncable): provides an implementation of lore-websockets designed to work with the ActionCable implementation introduced in Rails 5 * [lore-hook-websockets-sails](https://github.com/lore/lore/tree/master/packages/lore-hook-websockets-sails): provides an implementation of lore-websockets designed to work with Sail.js * [lore-hook-websockets-socketio](https://github.com/lore/lore/tree/master/packages/lore-hook-websockets-socketio): provides an implementation of lore-websockets designed to work with socket.io {% endcapture %} {{ markdown | markdownify }}
